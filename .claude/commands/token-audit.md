# /token-audit — придирчивый, но честный стюард дизайн-системы Disher

Сканируешь **ВСЕ** `scss`/`css` в `apps/food-calc/src/` и классифицируешь каждый: где
токенов нет — помечаешь; где есть — проверяешь на сырьё (raw) и канон. Собираешь список
**готов / не готов**.
**НИЧЕГО НЕ ПРАВИШЬ в app-коде.** Единственный файл, который создаёшь, — артефакт-отчёт.

## SCOPE — тупо по всем, БЕЗ git-фильтра

- Сканируй **все** `apps/food-calc/src/**/*.scss` + `**/*.css`. **НЕ** `git diff`, **НЕ**
  base-ref, **НЕ** «дельта кампании», **НЕ** commit-state. Закоммичено/нет — без разницы.
- `$1` (спека, дефолт `apps/food-calc/tds/typography-roles-system.md`) — **только канон-референс
  значений** ролей для N7/N-VAL. Scope она НЕ ограничивает.
- Чужой WIP параллельных сессий тоже сканится (он часть «всех файлов») — но в находках
  помечай кампанию, не приписывай одной.

## ПОРЯДОК (гейт ПЕРВЫМ, детектор ВТОРЫМ, семантика ТРЕТЬИМ)

1. **Гейт.** `cd .../apps/food-calc && npm run lint:style` и `... && npm run check:tokens`
   (одной строкой с `cd`, иначе персистнувший cwd сломает второй вызов). Вывод = ФАКТ по
   N4/N6 (существование токена + сырьё на gated-scss). Не перепроверяй глазами.
2. **Детектор (механика, по всем файлам).** Прогони `node scripts/token-sweep.cjs` из
   `apps/food-calc`. Он раскладывает каждый файл в корзину ГОТОВ / НЕ ГОТОВ / НЕТ ТОКЕНОВ /
   РЕЕСТР. Это костяк отчёта.
3. **Семантика (канон, по флагнутым + реестру).** На файлы из НЕ ГОТОВ и РЕЕСТР примени
   нормы N1/N2/N3/N5/N7/N-VAL — то, что гейт и детектор НЕ ловят (3-ярусность, полнота
   композита роли, bespoke-нейминг). Бюджет внимания — сюда.

## ДЕТЕКТОР — `scripts/token-sweep.cjs` (детерминированный, запусти как есть)

```bash
cd "<repo>/apps/food-calc" && node scripts/token-sweep.cjs
```

Вывод — 4 корзины + ⚠-секция `NON-BASE!`. **Костяк таблиц артефакта бери отсюда, не считай глазами.**

> Калибровка (зашита в скрипт; НЕ переизобретай — проверено эмпирически):
> - **bare-raw = реальный долг:** `#hex` / `rgb(a)` / `hsl(a)` на gated-свойстве
>   (`color` / `*-color` / `font-size` / `border-radius`) БЕЗ `var(` на строке.
> - **НЕ сырьё (легально, в ГОТОВ):** `var(--token, #raw)` (raw — фоллбэк, значение = токен) ·
>   `gradient(...)` в background (канон stripe-fork) · `box-shadow`/`background`-shorthand
>   (вне гейта) · raw в `--token: value` определении (ref-ярус).
> - **РЕЕСТР** = 8 primitive-файлов из `overrides[0].files` в `.stylelintrc.cjs`
>   (tokens / palette / surfaces / themes / design-variant-palettes / heading-font-variants /
>   mixin / index). Освобождены навсегда: raw тут = ref-ярус. Скрипт держит этот список —
>   **синхронь при правке конфига.**
> - **Теги НЕ ГОТОВ:** `[baseline]` (амнистия `stylelint-legacy-baseline.cjs`, убывает по
>   касанию) · `[disabled]` (file-level `/* stylelint-disable …declaration-strict-value */`) ·
>   `[css-UNGATED]` (`.css` вне глоба гейта `src/**/*.scss`) · `[NON-BASE!]` = сырьё в обычном
>   `.scss`, которого быть НЕ должно.
> - **⚠ NON-BASE! при зелёном гейте** = либо ложноположит. (`var(--t, raw)`-фоллбэк на
>   продолжении строки — regex не отсёк), либо реальная дыра. **Перепроверь строку глазами,
>   исправь вердикт в отчёте.** В норме секция пустая.

## ИСТОЧНИКИ ИСТИНЫ (сверяйся, НЕ по памяти)

- Спека (`$1`) — целевой маппинг ролей/значений. ПРИ КОНФЛИКТЕ с нормами ниже — спека побеждает.
- `apps/food-calc/src/shared/assets/style/{tokens,palette,themes,mixin}.scss` — реестр токенов.
- `apps/food-calc/docs/guidelines.md` — ИСТОЧНИК ЗНАЧЕНИЙ: шкала ×1.25, space 4/8, OKLCH-нейтраль,
  не-чисто-чёрный/белый, многослойные тонированные тени, вложенный радиус, табличные цифры, WCAG 4.5:1.
- Скилл /систематизация-дизайна + память (rename_noncanon_token_first, semantic_tokens_cti,
  no_per_surface_theming, design_unification).

## НОРМЫ (нарушение = строка в отчёте с цитатой)

- **N1** нейминг по СМЫСЛУ, не по компоненту: `--sys-<роль>` да; `--field-shadow`/`--bar-*`/`--nt-*` нет.
- **N2** ровно 2 яруса ref→sys: локальная прокладка на существующую sys-роль = 3-й ярус = FAIL.
- **N3** консумер ссылается на КАНОНИЧЕСКИЙ `--sys-*` напрямую, не на прокладку.
- **N5** без per-surface theming и без расширения легаси-бандлов `--bar-*`/`--seg-*`/`--field-*`.
- **N7** значение/роль = маппинг спеки; иначе FAIL (если не задокументировано как реклассификация).
- **N-VAL** значение ref-примитива на шкале guidelines.md; вне шкалы/ниже WCAG = долг, даже при зелёном гейте.
- [N4 существование + N6 сырьё на gated-scss — из гейта/детектора, не дублируем глазами.]

## ЛОВУШКИ (проверяй проактивно)

- нейминг по компоненту (`--field-shadow`/`--bar-*`/`--ax-*`/`--nt-*`) → первый ход: переименовать в `--sys-*` (не спрашивать старое значение).
- выдуманный токен вне шкалы (`--space-7/9/11`) = тихий 0 — гейт ловит только в статич. семействах.
- реинтродукция per-surface `[data-surface]`-override или 3-яруса `--control-*`/`--state-*` = FAIL.
- неполный композит роли: `var(--sys-text-title-size)` БЕЗ `-weight`/`-family` = не полная роль (кейс DailyAnalysisSection).
- decoupled-by-design (lemon CTA, `--seg-*` сегмент поиска, colored-press `--press-*`, `--row-cell-*` touch-floor, `--drawer-swipe-*`) — НЕ флагать.
- bare-raw в `[baseline]`/`[disabled]`-файле, тронутом по ДРУГОМУ поводу, — общий ratchet-долг, НЕ регресс волны: помечай как долг, не как FAIL кампании.

## ВЫХОД — ОБЯЗАТЕЛЬНО ДВА АРТЕФАКТА

### A. Файл-артефакт (главный результат)

`apps/food-calc/tds/ANALYSIS/token-audit-<YYYY-MM-DD>.md` (дата = `currentDate`). Структура:
1. **Header** (4 строки git-state до прозы, feedback_checkpoint_header_format): Last commit /
   Working tree / Scope (= «весь src/, N файлов») / Branch.
2. **Калибровка детектора** — короткий блок «что считаем bare-raw, что нет» (см. выше), чтобы читатель не спорил с цифрами.
3. **Гейт** — вывод `lint:style` + `check:tokens` + 1 фраза «почему зелёный при N не-готовых» (baseline-амнистия + `disabled` + `.css` вне глоба).
4. **Сводная таблица корзин:** `| Корзина | #файлов |` (ГОТОВ / НЕ ГОТОВ / НЕТ ТОКЕНОВ / РЕЕСТР / УДАЛЁН).
5. **НЕ ГОТОВ** — таблица `| raw | файл | строки | [тег] | замечание/кампания |`, строка на КАЖДЫЙ.
6. **РЕЕСТР** — вердикт по канону на каждый (3-ярусность, нейминг).
7. **ГОТОВ** — компактный список имён (по сырью чисто; прозы не надо). Помечай те, что ещё на легаси-bespoke (`--bar-*`/`--ax-*`/…) — кандидаты следующего яруса (категория, не per-файл FAIL).
8. **НЕТ ТОКЕНОВ / УДАЛЁН** — список имён.
9. **Канон-находки:** `файл:строка · N# · цитата · почему · правка (что→на что)` — только реальные, из семантического прохода.
10. **Итог.** Чистых FAIL/долга — числом. Приоритет №1 — назвать 1 файл. НЕ выдумывай findings ради объёма.

### B. Сводка в чат

Короткая: путь к артефакту + таблица корзин (#готов/#не-готов) + 1 приоритетная правка.
